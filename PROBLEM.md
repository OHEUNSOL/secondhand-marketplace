## 기능 요구사항

### 1. 회원가입 / 로그인

- **FastAPI에서 인증 로직 전체를 구현** (회원가입, 로그인, 토큰 발급)
- **이메일 + 비밀번호** 기반 로그인
- 비밀번호는 반드시 **bcrypt**로 해싱하여 저장
- **JWT 토큰** 기반 인증
  - 로그인 성공 시 Access Token 발급
  - 인증이 필요한 API에 토큰 검증 미들웨어(Depends) 적용
- Next.js 프론트엔드는 JWT를 저장하고, API 호출 시 Authorization 헤더에 담아 전송
- 회원 유형: **일반 사용자** / **관리자**
  - 관리자 계정은 시드 데이터로 미리 생성해두면 됩니다 (별도 가입 플로우 불필요)
- (선택) Refresh Token 구현 시 가산점

### 2. 판매 물품 등록

- 로그인한 사용자는 판매 물품을 등록할 수 있음
- 필수 입력: **상품명, 가격, 상품 설명, 카테고리, 상품 상태(새상품/중고)**
- 선택 입력: **상품 이미지** (최소 1장, 최대 5장)
  - 이미지 업로드 방식은 자유 (로컬 저장 or 이미지 URL 입력 방식 모두 가능)
- 물품 상태: **판매중 / 예약중 / 판매완료**
- 등록자는 자신의 물품을 **수정 / 삭제** 할 수 있음

### 3. 상품 목록 및 검색

- 전체 상품 목록 페이지
- **카테고리 필터** (예: 전자기기, 의류, 생활용품, 도서, 기타)
- **키워드 검색** (상품명, 설명에서 검색)
- **정렬**: 최신순, 가격 낮은순, 가격 높은순
- **페이지네이션** (방식 자유)

### 4. 상품 상세 페이지

- 상품 정보 전체 표시
- 판매자 정보 표시 (닉네임)
- **장바구니 담기** 버튼
- **바로 구매** 버튼

### 5. 장바구니

- 로그인한 사용자별 장바구니
- 상품 추가 / 수량 변경 / 삭제
- 장바구니 내 총 금액 합계 표시
- **선택 구매** (체크박스로 원하는 상품만 선택하여 구매 가능)

### 6. 구매

- 장바구니에서 선택 구매 또는 상세 페이지에서 바로 구매
- 실제 결제 연동은 **불필요** — 구매 버튼 클릭 시 바로 구매 완료 처리
- 구매 완료 시:
  - 해당 상품 상태를 **판매완료**로 변경
  - 구매 내역 저장 (구매자, 상품, 구매일시, 금액)
- **마이페이지**에서 구매 내역 확인 가능
- **판매자 마이페이지**에서 판매 내역 확인 가능

### 7. 관리자 기능

- 관리자 전용 페이지 (일반 사용자 접근 불가)
- **상품 목록 조회**: 전체 상품을 관리자 관점에서 조회
- **판매 중지 + 블라인드 처리**:
  - 관리자가 특정 상품을 "판매 중지" 처리할 수 있음
  - 판매 중지된 상품은 일반 사용자에게 **목록에서 표시되지 않거나** "블라인드 처리된 상품입니다"로 표시
  - 관리자는 블라인드 사유를 입력할 수 있음
- **블라인드 해제**: 관리자가 블라인드를 해제하면 다시 정상 노출

- **main 브랜치**: 프로덕션 브랜치 (안정된 코드만 머지)
